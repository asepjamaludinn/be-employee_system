# Leave Management System API

This is a RESTful API built for a technical test (Backend Engineer). It handles employee leave requests with role-based access control and automated leave quota validation.

## Features

- **OAuth 2.0 Authentication**: Login via Google using Laravel Socialite.
- **Token-Based API**: Secured using Laravel Sanctum.
- **Role-Based Access Control (RBAC)**: Distinguishes between `admin` and `employee`.
- **Automated Quota Validation**: Employees are granted 12 days of leave quota. The system automatically validates and deducts quotas upon admin approval.
- **Clean Architecture**: Separation of concerns using DTOs, Repositories, and Services.
- **Database Transaction**: Ensures data integrity during the approval and quota deduction process.

## Tech Stack

- **Framework**: Laravel 11
- **Database**: PostgreSQL (Hosted on Supabase)
- **Authentication**: Laravel Sanctum & Socialite

## Local Setup & Installation

Follow these steps to run the project on your local machine:

1.  **Clone the repository**
    ```bash
    git clone https://github.com/asepjamaludinn/be-employee_system.git
    cd be-employee_system
    ```
2.  **Install dependencies**

    ```Bash
    composer install
    ```

3.  **Environment Setup**

    ```Bash
    cp .env.example .env
    php artisan key:generate
    ```

    _Note: Ensure you configure the Supabase PostgreSQL credentials and Google OAuth keys in your .env file._

4.  **Database Migration & Seeding**
    Run the migrations and seed the database with initial Admin and Employee accounts:

    ```Bash
    php artisan migrate:fresh --seed
    ```

5.  **Serve the Application**

    ```Bash
    php artisan serve
    ```

    The API will be accessible at http://127.0.0.1:8000.

### Authentication

- `POST /api/login` - Conventional login.
- `GET /api/auth/google/redirect` - Get Google OAuth URL.
- `GET /api/auth/google/callback` - Handle Google OAuth callback.

### Leave Requests (Requires Bearer Token)

- `GET /api/leave-requests` - View leave requests (Admin sees all, Employee sees only their own).
- `POST /api/leave-requests` - Submit a new leave request (Employee).
- `PATCH /api/leave-requests/{id}/status` - Approve or reject a leave request (Admin only).
